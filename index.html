npm&bower

npm&bower

  • npm是一个包管理工具(node package manager)
  • 在项目跟文件夹下,使用命令npm install jquery 可以安装包,安装后会在node_modules中把指定的包下载到里面,并自动把其依赖的包也会下载
  • 在install之前使用命令npm init 创建一个package.json文件然后在下载包的时候使用命令npm install jquery –save 会在package.json中有一个dependencies节点记录本项目依赖的包,在下次使用的时候可以直接使用npm install命令去下载项目依赖的包.
  • 使用npm inistall jquery –save-dev会在package.json中有一个devDependencies节点记录项目开发阶段使用到的包,而跟项目运行没有关系.
  • 使用npm install jquery -g 去下载全局用到的包, 比如bower
  • 使用npm uninstall jquery 去卸载一个包
  • bower跟npm区别
    • bower主要用于Web页面开发时使用的包.
    • npm主要用于node开发时使用的包.

gulp

#gulp

  1. 在gulp跟文件中使用 npm init 创建 package.json文件
  2. 使用 npm install gulp –save-dev 添加项目在开发阶段对gulp的依赖并下载gulp,因为gulp只是在开发阶段使用,不在项目运行阶段使用,所有只需要在开发依赖中使用即可
  3. 在项目跟目录中创建gulpfile.js文件去设置需要用到的任务,文件名是固定的不可以自定义文件名
  4. 编译gulpfile设置各种任务

  5. gulp执行各项任务需要依赖的包

    • gulp-less 编译less的工具
    • gulp-concat 合并代码
    • gulp-uglify 压缩js代码
    • gulp-cssnano 压缩css
    • gulp-htmlmin 压缩html
    • browser-sync 自动化同步服务器

  6. 编译gulpfile,把相应的任务写好以后,在命令行中输入 gulp style(任务名称)去执行相应任务

git&github

#git&github

  1. cd 到当前目录
  2. git init 初始化一个本地仓库
  3. git status 查看本地仓储的文件状态
  4. git add . 将所有没有被跟踪的文件添加到跟踪列表
  5. 添加git的忽略
    • 在代码库跟文件夹添加一个.gitignore
    • 把要忽略的文件写在.gitignore中
  6. git commit -m “提交信息” 把本地变化提交到本地的仓库文件夹归档
  7. git diff 查看本地文件夹和git中保存的最新的文件比较
  8. git log 查看git提交日志
  9. git reset –hard 2d2250(log的hash值) 把代码追溯到指定版本
  10. git help 查看git的左右命令

##github

###基本应用

  1. 首先链接远程仓库 git remote add origin https://github.com/xinqidianjiaoyu/git-demo.git(github链接网址)
    • 首次链接远程仓库会需要输入github用户名和密码用来验证用户是否有权限推送或者拉取代码
    • 只有首次需要执行此操作,以后项目结构会自动把网址分析为origin
  2. 网远程仓库推送代码 git push -u origin master
    • 此处的 -u 是用流的方式往github上推送代码,会快一点
    • origin 就是远程仓库的地址
    • master 表示是远程代码仓库的主分支
    • 在每次push之前要先pull一下,防止代码冲突
  3. 从远程仓库拉取代码 git pull origin master
    • 从远程仓库拉取代码的时候不需要加 -u 参数. 加了会报错

github创建分支

  1. 查看项目所有分支和当前分支 git branch
  2. 新建分支 git branch branchName
  3. 切换分支 git checkout branchName

github托管页面(创建 gh-pages 分支)

  1. 把项目添加到 gh-pages 分支就可以在线预览文件
  2. 项目地址为 xinqidianjiaoyu.github.io/productName (项目名称是区分大小写的)

zhang

call()

call(obj, arg1, arg2);
  1. 把调用call()函数的方法内的this指针指向call()里的第一个参数
  2. 也就是说调用call()的函数的实现给了call()的第一个参数

  3. call()里的第一个参数是调用call()的对象,其余参数也要是对象,从第二个参数开始后的参数会当成参数传进call()的方法调用里

    (function () {
        var obj = {
        name: "jack"
    };
    obj.showM = function () {
        console.log(this, arguments[0].name, arguments[1].age);
        //Object {name: "kai"} "kai" 20
    };

    var obj1 = {
           name: "kai"
    };
    var obj2 = {
    age: 20
    };

    obj.showM.call(obj1, obj1, obj2); 
})()

  4. 在这个方法中把obj.showM的方法实现,交给了obj1, 并且分别把obj1,和obj2当做参数传入了obj1调用obj.showM方法的过程中

apply()

apply(obj, [arg1, arg2])
  1. apply()的用法跟call()基本一致,只是语法稍有不同
  2. call()中的参数都为对象,且从第二个对象开始就是会当做参数传递
  3. apply()中只有两个参数,第一个参数为调用方法的对象,第二个参数是一个数组,执行时会把数组中的每个元素当做参数传递到方法的调用过程中
function bb() {
    console.log()this, arguments); 
    //{name: "jackChen, age: 24}   [hao a zhang kai](伪数组 有 0,1,2,3,callee,length属性)
}

var kai = {
    name: "jackChen",
    age: 24
};

var obj = {
    name: "jssts"
}

obj.cc = function () {
    return bb.apply(this, Array.prototype.slice.call(arguments, 1));
}

obj.cc.call(kai, "ni","hao","a","zhang","kai");

  • 在此案例中首先要明白Array.prototype,slice(start,end)方法

    • 这个方法本身为数组的截取方法,是从start角标到end角标,包含start不包含end [start end),而且返回的是一个截取好的数组
    • 如果不传end默认是从start开始截取到数组末尾
    • 而通过调用Array.prototype.slice.call(arguments, 1)可以把是伪数组的auguments弄成一个数组返回,而后边的1则是从角标1截取到数组末尾
    • arguments 中的length属性决定数组的长短,而且auguments中的属性必须是0–> 根据length去生成原有的角标的数组

  • 根据 Array.prototype.slice.call(arguments, 1) 方法返回的是[“hao”, “a”, “zhang”, “kai”]

  • 此时obj.cc方法中的this是kai这个对象
  • obj.cc中的方法中的返回值是执行bb方法,bb.apply()中的this是kai这个对象,第二个参数的数组中的每一项会当成参数传进bb的方法中执行
  • 也就是说最后的结果就是kai这个对象在调用bb的方法实现,并且带有参数,所以打印的结果是this是kai这个对象,arguments是一个伪数组

hexo搭建博客

#hexo

##hexo搭建本地博客

  1. 全局安装hexo npm install hexo -g
  2. 创建一个blog文件夹,在blog文件夹中执行 hexo init 初始化hexo , 会自动下载hexo的依赖
  3. 执行命令生成静态html文件 hexo generate
  4. 执行 hexo server 会生成一个服务器 并监听4000端口 输入 http://localhost:4000 即可访问本地博客

配置Guthub

  1. 建立Repository 名称必须为 xinqidianjiaoyu(userName).github.io

  2. 然后打开blog文件夹下面的 _config.yml 文件 翻到最下面, 配置deploy

    deploy:

     type: git

     repo: https://github.com/leopardpan/leopardpan.github.io.git

     branch: master

  3. 安装 hexo-deployer-git 插件(让文件支持git上传)

  4. 最后执行命令 hexo deploy 把本地的public中的静态博客部署到github的仓库中
  5. 在浏览器中输入 https://xinqidianjiaoyu.github.io 就可访问github上的博客了

###每次使用指令步骤

每次部署的步骤，可按以下三步来进行。

hexo clean    清空public文件夹

hexo generate    根据 source/_posts/*.md文件生成public文件夹和对应的网页文件

hexo deploy  把本地public文件夹中的网站资源上传到github服务器中

通过 https://xinqidianjiaoyu.github.io 就可以访问网站博客了

一些常用命令：

hexo new"postName" #新建文章

hexo new page"pageName" #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）

hexo deploy #将.deploy目录部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本
