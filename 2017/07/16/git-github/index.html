git&github

#git&github

  1. cd 到当前目录
  2. git init 初始化一个本地仓库
  3. git status 查看本地仓储的文件状态
  4. git add . 将所有没有被跟踪的文件添加到跟踪列表
  5. 添加git的忽略
    • 在代码库跟文件夹添加一个.gitignore
    • 把要忽略的文件写在.gitignore中
  6. git commit -m “提交信息” 把本地变化提交到本地的仓库文件夹归档
  7. git diff 查看本地文件夹和git中保存的最新的文件比较
  8. git log 查看git提交日志
  9. git reset –hard 2d2250(log的hash值) 把代码追溯到指定版本
  10. git help 查看git的左右命令

##github

###基本应用

  1. 首先链接远程仓库 git remote add origin https://github.com/xinqidianjiaoyu/git-demo.git(github链接网址)
    • 首次链接远程仓库会需要输入github用户名和密码用来验证用户是否有权限推送或者拉取代码
    • 只有首次需要执行此操作,以后项目结构会自动把网址分析为origin
  2. 网远程仓库推送代码 git push -u origin master
    • 此处的 -u 是用流的方式往github上推送代码,会快一点
    • origin 就是远程仓库的地址
    • master 表示是远程代码仓库的主分支
    • 在每次push之前要先pull一下,防止代码冲突
  3. 从远程仓库拉取代码 git pull origin master
    • 从远程仓库拉取代码的时候不需要加 -u 参数. 加了会报错

github创建分支

  1. 查看项目所有分支和当前分支 git branch
  2. 新建分支 git branch branchName
  3. 切换分支 git checkout branchName

github托管页面(创建 gh-pages 分支)

  1. 把项目添加到 gh-pages 分支就可以在线预览文件
  2. 项目地址为 xinqidianjiaoyu.github.io/productName (项目名称是区分大小写的)
